# Домашняя работа

## Урок №1 — Что такое контроль версий. Установка Git и первая настройка

**Срок сдачи:** к уроку 2

### Задание

Задание на повторение (письменно в тетради или в файле):

1. Своими словами объясни, что такое система контроля версий и какую проблему она решает (2–3 предложения).
2. Напиши, в каком году и кто создал Git.
3. Выпиши три команды, которые мы использовали на уроке, и подпиши, что делает каждая (`git --version`, `git config --global user.name`, `git config --list`).
4. Если дома есть компьютер — установи на нём Git и редактор VS Code, затем выполни первую настройку Git (имя и e-mail) самостоятельно.

### Критерии выполнения

- [ ] Дано определение системы контроля версий и описана решаемая ею проблема.
- [ ] Указаны год создания Git и его автор.
- [ ] Выписаны три команды с пояснением, что делает каждая.
- [ ] (При наличии домашнего компьютера) Git и VS Code установлены, выполнена первичная настройка Git.

## Урок №2 — Первый репозиторий: init, add, commit, status, log

**Срок сдачи:** к уроку 3

### Задание

Задание на повторение (письменно в тетради или в файле):

1. Выпиши по порядку пять команд урока (`git init`, `git status`, `git add`, `git commit`, `git log`) и подпиши, что делает каждая.
2. Своими словами объясни, что такое индекс и зачем нужен `git add` перед коммитом (2–3 предложения).
3. Нарисуй или опиши путь файла из трёх ступеней: рабочая папка, индекс, коммит.
4. Если дома есть компьютер с Git — создай репозиторий `my-home-repo`, добавь в него один файл и сделай первый коммит.

### Критерии выполнения

- [ ] Выписаны пять команд урока с пояснением, что делает каждая.
- [ ] Объяснено, что такое индекс и зачем нужен `git add` перед коммитом.
- [ ] Изображён или описан путь файла из трёх ступеней (рабочая папка → индекс → коммит).
- [ ] (При наличии домашнего компьютера) Создан репозиторий `my-home-repo` с первым коммитом.
