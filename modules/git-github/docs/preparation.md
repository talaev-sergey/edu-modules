# Введение в модуль: От Git до Github

## Общая информация

| Параметр              | Значение         |
| --------------------- | ---------------- |
| Модуль                | От Git до Github |
| Возраст учащихся      | 12–14 лет        |
| Количество уроков     | 8                |
| Продолжительность     | 120 мин (один урок) |

---

## О модуле

Модуль знакомит школьников с системой контроля версий Git и платформой GitHub.
Ученики проходят путь от первого локального репозитория на своём компьютере до
совместной работы над проектом в интернете и публикации результата как сайта на
GitHub Pages. Опыт работы с Git не требуется — модуль рассчитан на новичков.

Эти навыки — основа повседневной работы любого программиста: они позволяют не
терять свои наработки, безопасно экспериментировать и работать над проектом в
команде.

---

## Цели модуля

По завершении модуля учащийся должен:

- **Знать:** что такое система контроля версий и какие задачи она решает; базовые
  понятия Git (репозиторий, коммит, история, ветка) и GitHub (удалённый репозиторий,
  Pull Request, Issues).
- **Уметь:** создавать локальный репозиторий и фиксировать изменения коммитами;
  просматривать историю и возвращаться к нужной версии; создавать ветки и выполнять
  слияние; публиковать проект на GitHub и синхронизировать его (push, pull, clone);
  участвовать в командной работе через Pull Request и Issues; публиковать результат
  как сайт на GitHub Pages.
- **Иметь опыт:** ведения репозитория от первого коммита до публикации страницы
  на общем сайте класса (GitHub Pages).

## Предварительные требования

Что учащийся должен знать и уметь перед началом модуля:

- запускать программы и открывать браузер в Windows 10/11;
- уверенно пользоваться клавиатурой и мышью;
- иметь рабочий e-mail или возможность завести его (понадобится с урока 5).

Опыт работы с Git, командной строкой или программированием не требуется.

## Установка и настройка среды

Пошаговая инструкция по подготовке рабочих мест к модулю:

1. Заранее проверьте на каждом ПК наличие прав администратора — без них Git и VS Code не установятся. При необходимости согласуйте установку с системным администратором школы.
2. Убедитесь, что открываются сайты git-scm.com, code.visualstudio.com и github.com.
3. На урок 1 ученики самостоятельно скачивают и устанавливают Git и VS Code, затем выполняют первичную настройку Git (имя и e-mail). На случай проблем с интернетом держите установщики скачанными заранее.
4. К уроку 5 каждый ученик должен завести аккаунт на GitHub (онлайн, без установки) с подтверждённым e-mail.
5. Прогоните установку на одном ПК заранее и запишите ожидаемый номер версии Git — это поможет быстро проверять результат у учеников.

## Рекомендации преподавателю

- Подготовьте машины заранее: права администратора и скачанные установщики снимают большую часть проблем урока 1.
- Проследите, чтобы у каждого ученика был подтверждённый e-mail до урока 5 — без него не зарегистрировать аккаунт GitHub.
- Модуль идёт в режиме теории «Б»: теории мало, она вплетена в практику короткими блоками «мини-теория → задание». Не затягивайте объяснения, быстрее переходите к действиям.
- Работайте по принципу «Я показываю — делаем вместе — делаешь сам»: каждый новый шаг сначала демонстрируйте на проекторе.
- Учитывайте слабые навыки работы с Windows: называйте элементы интерфейса точно, давайте полные пути, разбивайте действия на отдельные шаги.
- Домашние задания — на повторение пройденного; они закрепляют материал, но не вводят новый.
- Для тех, кто справляется быстрее, держите наготове дополнительные задания (в разделе «Дополнительные задания» в конце каждого урока).
