# Гайд: скиллы Codex для подготовки уроков

Этот гайд — для команды методистов. Объясняет, как пользоваться двумя скиллами
в десктопном Codex на Windows. Технических знаний не требуется: вы пишете запрос
обычным текстом, скилл подключается сам по ключевым фразам.

---

## Что такое скилл

Скилл — это готовая инструкция для Codex по конкретной задаче. Лежат они в папке
`.codex/skills/` внутри проекта. У нас их два:

| Скилл | Что делает |
|-------|-----------|
| **it-metodist** | Пишет черновик урока — чистый Markdown в папку `content/`. |
| **mkdocs-programmer** | Превращает черновик в страницу сайта MkDocs и проверяет вёрстку. |

Порядок работы всегда один: **сначала `it-metodist` (черновик) → потом
`mkdocs-programmer` (вёрстка на сайт)**.

---

## Подготовка (один раз)

1. Откройте десктопный Codex.
2. Откройте папку проекта (File → Open Folder или перетащите папку).
3. Убедитесь, что проект помечен как доверенный (trusted) — Codex спросит при первом
   открытии, ответьте «Trust».
4. Скиллы подхватываются автоматически: ничего включать вручную не нужно.

> Все команды внутри скиллов уже написаны и для Windows, и для Linux. На вашем
> Windows-десктопе Codex сам определит систему и возьмёт нужные команды
> (PowerShell / cmd).

## Скилл 1 — it-metodist (черновик урока)

### Как запустить

Напишите в чат Codex любую из фраз (подставьте номер урока):

- `напиши черновик урока 3`
- `составь конспект урока 3`
- `подготовь методичку урока 3`
- `напиши урок по теме «...»`

### Что произойдёт

1. Codex прочитает стратегию модуля из `content/strategy.md`.
2. Если чего-то не хватает (тема, возраст, длительность, номер урока, название модуля) —
   **задаст все вопросы одним сообщением**. Ответьте на них.
3. Создаст файл `content/lesson_NN.md` — готовый черновик урока на русском.
4. Выдаст в чат отчёт: тайминг по этапам, список нужных скриншотов, заметки для методиста.

### Что нужно от вас

- Заранее заполните `content/strategy.md` (стратегия модуля) — чем подробнее, тем меньше вопросов.
- Обязательные поля, без них работа не начнётся: **номер урока, тема, возраст, длительность,
  название модуля/курса**.
- В черновике будут метки вида `[СКРИНШОТ: что показать]`. Скриншоты вы делаете сами
  вручную позже — скилл вёрстки вставит их в нужные места.

## Скилл 2 — mkdocs-programmer (вёрстка на сайт)

### Как запустить

После того как черновик готов и проверен, напишите:

- `верстай урок 3` — собрать один урок
- `верстай все уроки` — собрать все черновики из `content/`
- `запусти сайт` — поднять локальный сайт для просмотра
- `проверь сайт` — сделать скриншот и проверить вёрстку
- `останови сайт` — выключить локальный сервер

### Что произойдёт

1. Codex возьмёт черновик `content/lesson_NN.md`.
2. Создаст готовую страницу `docs/lessons/lesson_NN.md` с правильным оформлением
   (заметки, вкладки, слайды).
3. Добавит урок в меню сайта (`mkdocs.yml`).
4. Запустит сайт локально и сделает скриншот, чтобы убедиться, что всё отображается верно.

### Картинки и скриншоты

- Положите изображения в папку `content/images/` — скилл сам скопирует их на сайт.
- Если для метки `[СКРИНШОТ: ...]` картинки ещё нет, скилл оставит пометку-напоминание
  и перечислит её в отчёте. Ничего не пропадёт молча.

## Типовой сценарий целиком

```
1. Заполнили content/strategy.md
2. «напиши черновик урока 5»        → it-metodist создаёт content/lesson_05.md
3. Прочитали черновик, при желании поправили
4. Сделали нужные скриншоты, положили в content/images/
5. «верстай урок 5»                 → mkdocs-programmer собирает страницу сайта
6. «запусти сайт»                   → смотрим результат в браузере (http://127.0.0.1:8000)
7. «останови сайт»                  → когда закончили
```

## Особенности Windows

- Скиллы рассчитаны на учеников 6–17 лет на Windows 10/11 со слабыми навыками ПК —
  в уроках всегда полные пути и точные названия элементов интерфейса.
- Если PowerShell ругается на запуск скриптов (`Activate.ps1 blocked`), один раз
  выполните в PowerShell:
  `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
- Если порт 8000 занят (сайт не запускается), Codex сам остановит старый процесс;
  при ручной необходимости: `netstat -ano | findstr :8000` → `taskkill /PID <номер> /F`.

## Частые вопросы

**Скилл не подключился.** Проверьте, что открыта именно папка проекта
и что в запросе есть ключевая фраза («черновик урока N», «верстай урок N»).

**Codex задаёт много вопросов.** Значит, в `strategy.md` не хватает данных. Ответьте
один раз — или заполните стратегию подробнее.

**Можно ли сразу верстать без черновика?** Нет. Сначала `it-metodist` делает черновик
в `content/`, потом `mkdocs-programmer` его верстает.

**Где лежат сами скиллы?** В `.codex/skills/it-metodist/` и `.codex/skills/mkdocs-programmer/`.
Править их не нужно — это рабочие инструкции, а не материалы урока.
