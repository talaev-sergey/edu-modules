## Учебная программа ООП на C#.docx

Мы работаем над проектом: «Написание учебных программ для детей и подростков».
Текущая тема: «Введение в ООП на C#».
Возраст: от 12 лет.
Уровень: дети — полные новички в ООП.
Среда практики: Visual Studio 2022 Community Edition.
Язык: C#.
Цель курса: дать детям понятное первое представление об объектно-ориентированном программировании и подготовить их к дальнейшему переходу в Unity.
Общий стиль работы:
Не писать огромные стены теории или кода, пока я явно не попрошу.
Перед созданием материала перепроверять себя.
Писать для полных новичков: с нуля, по шагам, понятными словами.
Использовать простые аналогии: чертеж и предмет, персонаж в игре, собака/кот, герой/враг/предмет.
Теория не должна быть сухой: обязательно должны быть вопросы группе, практика, небольшие задачи.
В конце каждого занятия обязательно домашнее задание.
В местах, где нужны изображения, скриншоты или схемы, ставить специальные пометки прямо внутри сценария, а не отдельным списком.
Главное правило по DOCX-документам: Нужно делать документы в ТОЧНО ТАКОМ ЖЕ визуальном и структурном формате, как утвержденные файлы:

Если в новом чате есть возможность загрузить файл-шаблон, нужно использовать именно его как основу. Не пересобирать оформление вручную. Нужно брать утвержденный DOCX как шаблон, менять только содержимое занятия и сохранять:

Занятие 4. Инкапсуляция: защищаем данные объекта Цель: объяснить, почему нельзя давать любому коду свободно менять внутренние данные объекта. Обязательные темы:

список объектов;

virtual;
override;
метод Attack();

1.1. Теория: объясняем с нуля
Преподаватель объясняет: чтобы писать программы, нам нужна не просто клавиатура и не просто текстовый файл. Нам нужна специальная программа для программиста — среда разработки. По-английски ее часто называют IDE. Visual Studio — это большая “мастерская программиста”, где можно писать код, запускать программу, видеть ошибки и исправлять их.
🟨 ИЛЛЮСТРАЦИЯ: показать аналогию “мастерская”: инструменты, рабочий стол, заготовка проекта. Подписать: “Visual Studio = мастерская для программы”.
Дети должны понять простую мысль: C# — это язык, на котором мы объясняем компьютеру задачу, а Visual Studio — место, где мы удобно пишем и запускаем этот код.
🟨 СКРИНШОТ: показать официальную страницу загрузки Visual Studio Community. Выделить кнопку Download / Скачать для Community Edition.
После запуска установщика открывается Visual Studio Installer. В нем нужно выбрать, какие инструменты установить. Для нашего курса нужен набор для C# и .NET. Преподаватель показывает рабочую нагрузку “.NET desktop development”.
🟥 ВАЖНО НА СКРИНШОТЕ: выделить в Visual Studio Installer пункт “.NET desktop development”. Отдельно сказать: “Не выбираем все подряд, чтобы не ставить лишнее”.

2.1. Теория: метод, который можно заменить
Преподаватель объясняет: иногда базовый класс дает общий метод, но разрешает дочерним классам изменить его поведение. Для этого в C# у метода родителя пишут слово virtual.
Простая формулировка: virtual означает “этот метод можно переопределить в дочернем классе”.
Если дочерний класс хочет заменить поведение родителя, он пишет override. Простая формулировка: override означает “я делаю свою версию этого метода”.
