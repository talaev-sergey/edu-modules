# Стратегия модуля

## Курс и модуль

- Название курса: Программирование на C#
- Название модуля: Введение в ООП

## Аудитория

- Возраст учеников: 12-15 лет
- Уровень подготовки: базовый
- Особенности группы: ученики уже умеют запускать простые C#-программы, создавать консольные проекты, использовать переменные, условия, циклы, массивы или списки и простые методы. Навыки работы с Windows могут быть слабыми: в уроках нужны пошаговые инструкции для Проводника, запуска IDE, открытия папки проекта и запуска программы.

## Структура модуля

- Сколько уроков в модуле: 6
- Продолжительность одного урока (мин): 120 мин.
- Список уроков:
  1. Классы и объекты: создаем первую модель
  2. Поля, свойства и методы: учим объект хранить данные и действовать
  3. Конструктор и состояние объекта: создаем объекты правильно
  4. Инкапсуляция: защищаем данные и управляем изменениями
  5. Наследование и переопределение: строим семейство классов
  6. Итоговый мини-проект: ООП-приложение с несколькими классами

## Цели модуля

- Чему ученики научатся по итогам модуля: создавать простые классы на C#, создавать объекты через `new`, добавлять поля, свойства и методы, использовать конструкторы, хранить состояние объекта, применять инкапсуляцию через свойства и методы управления данными, использовать модификаторы доступа `public` и `private`, строить простую иерархию наследования и объяснять, когда ООП помогает сделать программу понятнее.
- Практический результат: консольное C#-приложение с несколькими связанными классами. Рекомендуемая тема проекта: каталог игровых персонажей, питомцев, книг, фильмов или товаров. В проекте должны быть классы, объекты, свойства, методы, конструкторы, проверка данных и минимум один пример наследования.

## Инструменты и среда

- Программы и инструменты, которые используются на уроках: Windows 10/11, Проводник, .NET SDK 8 или новее, Visual Studio Code с расширением C# Dev Kit или Visual Studio Community, терминал Windows или командная строка Windows при необходимости.
- Нужна ли установка чего-либо: да. На первом уроке преподаватель проверяет наличие .NET SDK и редактора. Если .NET SDK не установлен, используется заранее подготовленная инструкция установки для Windows. В уроках важно отдельно проверять, что ученики открыли именно папку проекта, а не отдельный файл, и запускают проект командой `dotnet run` или кнопкой запуска в IDE.

## Режим теории

Выбери режим для модуля (можно указать разный для разных уроков):

**Режим А** - стандартный. Теория идет единым блоком в начале урока, занимает не более 25% времени.
Подходит для большинства тем: инструменты, интерфейсы, базовые концепции.

**Режим Б** - теория вплетена в практику. Вводный блок теории 5-10 мин, затем по ходу
практической работы вставляются короткие блоки «мини-теория -> задание».
Подходит, когда без объяснения нельзя сделать следующий шаг: алгоритмы, сети,
системы счисления, архитектура программ.

- Режим теории для модуля: режим А для уроков 1-4, режим Б для уроков 5-6. В уроках 5-6 ученикам проще усваивать наследование, `virtual`, `override` и проектирование через короткие практические шаги, а не через большой теоретический блок.

## Детализация уроков

### Урок 1. Классы и объекты: создаем первую модель

- Цель урока: к концу урока ученики смогут создать класс C#, создать 2-3 объекта этого класса через `new` и вывести значения их данных в консоль.
- Ключевые понятия: класс, объект, экземпляр, поле, оператор `new`.
- Практика: класс `Hero`, `Pet` или `Book`; создание нескольких объектов; вывод информации через `Console.WriteLine`.
- Самостоятельная работа: создать свой класс по выбранной теме и минимум три объекта.
- Домашнее задание: повторение. Придумать 5 сущностей из реальной жизни и записать для каждой возможные данные, которые можно хранить в объекте.

### Урок 2. Поля, свойства и методы: учим объект хранить данные и действовать

- Цель урока: к концу урока ученики смогут добавить в класс свойства и методы, а затем вызвать методы для разных объектов.
- Ключевые понятия: поле, свойство, метод, вызов метода, поведение объекта.
- Практика: свойства `Name`, `Level`, `Health`; методы `ShowInfo()`, `LevelUp()`, `Rename()`, `IsStrong()` или похожие по теме проекта.
- Самостоятельная работа: добавить 2-3 свойства и 2-3 метода в класс из урока 1 и проверить работу на нескольких объектах.
- Домашнее задание: повторение. Описать словами, чем свойство отличается от метода, и привести 3 примера.

### Урок 3. Конструктор и состояние объекта: создаем объекты правильно

- Цель урока: к концу урока ученики смогут написать конструктор, передать начальные значения при создании объекта и изменять состояние объекта через методы.
- Ключевые понятия: конструктор, параметр конструктора, начальное состояние, изменение состояния.
- Практика: перенос начальных значений в конструктор; создание объектов одной строкой через `new Hero("Alex", 1, 100)`; методы, которые меняют значения свойств.
- Самостоятельная работа: доработать класс так, чтобы все важные данные задавались через конструктор.
- Домашнее задание: повторение. Создать небольшой класс по аналогии и подготовить 2 объекта с разными начальными значениями.

### Урок 4. Инкапсуляция: защищаем данные и управляем изменениями

- Цель урока: к концу урока ученики смогут объяснить, зачем скрывать часть данных внутри класса, и реализовать простую проверку значений через свойства или методы.
- Ключевые понятия: инкапсуляция, `public`, `private`, свойство, проверка данных, внутреннее состояние.
- Практика: закрытые поля или приватные сеттеры; методы `SetHealth()`, `AddScore()`, `SpendEnergy()` или похожие; запрет отрицательных значений; понятные сообщения об ошибках.
- Самостоятельная работа: добавить в свой класс минимум две проверки данных и скрыть прямое изменение важных значений.
- Домашнее задание: повторение. Найти в своем проекте 3 места, где нужна проверка данных, и описать правило проверки.

### Урок 5. Наследование и переопределение: строим семейство классов

- Цель урока: к концу урока ученики смогут создать базовый класс и 2 дочерних класса, а также переопределить метод в дочернем классе через `virtual` и `override`.
- Ключевые понятия: наследование, базовый класс, дочерний класс, `virtual`, `override`, переопределение метода.
- Практика: базовый класс `Character`, `Item` или `Animal`; дочерние классы с отличающимся поведением; сравнение дублирования кода до и после наследования.
- Самостоятельная работа: добавить в свой проект базовый класс и минимум два класса-наследника.
- Домашнее задание: повторение. Нарисовать простую схему классов своего проекта: базовый класс, наследники, главные свойства и методы.

### Урок 6. Итоговый мини-проект: ООП-приложение с несколькими классами

- Цель урока: к концу урока ученики смогут собрать консольное C#-приложение из нескольких классов и продемонстрировать работу объектов через сценарий запуска.
- Ключевые понятия: проектирование классов, связь объектов, сценарий программы, демонстрация результата.
- Практика: планирование классов; сборка проекта; тестовый запуск; исправление ошибок компиляции и логических ошибок.
- Самостоятельная работа: завершить мини-проект и подготовить короткую демонстрацию.
- Домашнее задание: проект. Доработать оформление вывода и добавить одну новую возможность по выбору.

## Логика нарастания сложности

- Урок 1: ученики видят разницу между описанием сущности и конкретным объектом.
- Урок 2: объект получает данные и поведение, программа становится менее набором отдельных переменных и методов.
- Урок 3: создание объектов становится аккуратным и повторяемым через конструктор.
- Урок 4: ученики учатся не менять данные хаотично, а управлять изменениями через свойства и методы.
- Урок 5: появляется повторное использование кода через наследование.
- Урок 6: все идеи собираются в небольшой самостоятельный проект.

## Домашние задания

- Предусмотрены ли домашние задания: да
- Если да - какого типа: повторение после уроков 1-5, проектная доработка после урока 6. Домашние задания должны быть короткими: 15-25 минут, без установки новых программ дома.

## Критерии итогового результата модуля

1. В проекте есть минимум 3 класса.
2. Объекты создаются через конструктор и оператор `new`.
3. В классах есть свойства и методы, которые выводят информацию или изменяют состояние объекта.
4. Есть минимум две проверки данных внутри свойств или методов.
5. Используются модификаторы доступа `public` и `private`.
6. Есть минимум один пример наследования.
7. Код запускается на Windows без дополнительных библиотек через IDE или команду `dotnet run`.
8. Ученик может устно объяснить, какие классы есть в проекте и за что отвечает каждый класс.

## Методические акценты

- На каждом уроке сначала показывать полный рабочий пример, затем давать задание на частичное изменение, затем самостоятельное задание по аналогии.
- Не вводить много терминов за один раз. В начале каждого урока достаточно 3-4 ключевых понятий.
- Постоянно связывать ООП с реальными объектами: персонаж, питомец, книга, товар, аккаунт, заказ.
- Следить, чтобы ученики не путали класс и объект. Полезная формула: класс - чертеж, объект - конкретная вещь по этому чертежу.
- Отдельно объяснять синтаксис C#, который часто мешает новичкам: фигурные скобки, точка с запятой, заглавные буквы в именах свойств и методов, сообщения об ошибках компиляции.
- Учитывать слабые навыки Windows: отдельно проговаривать, где находится папка проекта, как открыть ее в редакторе, как запустить программу, что делать, если открыли не ту папку или терминал находится не в папке проекта.
- В самостоятельных заданиях разрешать разные темы проекта, чтобы ученики могли выбрать знакомую предметную область.

## Дополнительно

- Связь с предыдущим или следующим модулем: модуль опирается на базовые темы C#: переменные, условия, циклы, массивы или списки, простые методы, ввод и вывод в консоли. После модуля логично переходить к работе с файлами, коллекциями, LINQ на базовом уровне, простым текстовым интерфейсам, небольшим играм или GUI-проектам.
- Особые требования или ограничения: не использовать внешние библиотеки. Не перегружать уроки UML и сложной терминологией. Все проекты должны запускаться локально на Windows 10/11.
