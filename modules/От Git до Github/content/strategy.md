# Стратегия модуля

## Курс и модуль

- Название курса: От Git до Github
- Название модуля: От Git до Github

## Аудитория

- Возраст учеников: 12-14
- Уровень подготовки (нет опыта / базовый / средний): нет опыта
- Особенности группы (если есть):

## Структура модуля

- Сколько уроков в модуле: 8
- Продолжительность одного урока (мин): 120 мин.
- Список уроков (номер и тема каждого):
  1. Что такое контроль версий. Установка Git и первая настройка
  2. Первый репозиторий: init, add, commit, status, log
  3. История изменений: просмотр, отмена, возврат к версии
  4. Ветки: создание, переключение, слияние
  5. GitHub: аккаунт, удалённый репозиторий, push и pull
  6. Клонирование и работа с чужими проектами: clone, fork
  7. Командная работа: Pull Request, Issues, обзор кода
  8. Итоговый практикум: общий сайт класса на GitHub Pages

## Цели модуля

- Чему ученики научатся по итогам модуля:
  - создавать локальный репозиторий и фиксировать изменения коммитами;
  - просматривать историю изменений и возвращаться к нужной версии;
  - работать с ветками: создавать, переключаться, выполнять слияние;
  - публиковать проект на GitHub и синхронизировать его (push/pull, clone);
  - участвовать в командной работе через Pull Request и Issues;
  - публиковать свою страницу на общем сайте класса (GitHub Pages).
- Какой практический результат получат (проект, программа, навык): навык

## Инструменты и среда

- Программы и инструменты, которые используются на уроках: Git (включая Git Bash),
  командная строка / терминал, браузер, редактор кода VS Code, аккаунт на GitHub
- Нужна ли установка чего-либо — и если да, на каком уроке:
  - урок 1 — установка Git и первичная настройка (имя, e-mail), установка VS Code;
  - урок 5 — регистрация аккаунта на GitHub (онлайн, без установки)

## Режим теории

Выбери режим для модуля (можно указать разный для разных уроков):

**Режим А** — стандартный. Теория идёт единым блоком в начале урока, занимает не более 25% времени.
Подходит для большинства тем: инструменты, интерфейсы, базовые концепции.

**Режим Б** — теория вплетена в практику. Вводный блок теории 5–10 мин, затем по ходу
практической работы вставляются короткие блоки «мини-теория → задание».
Подходит, когда без объяснения нельзя сделать следующий шаг: алгоритмы, сети,
системы счисления, архитектура программ.

- Режим теории для модуля: Б
